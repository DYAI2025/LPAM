# Product Canvas: LPAM Transferable Deployable Kit

Status: draft
Feature Slug: lpam-deployable-kit

| Section | ID | Value | Source Type | Source |
|---|---|---|---|---|
| Problem | CAN-001 | The working "Hermes + gbrain + voice" setup is bespoke and un-transferable: hardcoded `/root/...` paths, a real session token inlined in a systemd unit, no `.env.example`, an `install.sh` ↔ `.gitmodules` contradiction (cloned the wrong upstream + clobbered the submodule), no frontend clone, and manual one-off systemd/tunnel/voice wiring. A stranger could not reproduce it. | EXPLICIT | SRC-001, SRC-005 |
| Users / Customers | CAN-002 | Operators self-hosting Hermes/MCP agents who own a VPS (Ubuntu/AMD64) and a local Mac, joined by Tailscale. | EXPLICIT | SRC-001, SRC-003 |
| Value Promise | CAN-003 | One repo → reproducible deploy of the gbrain memory stack + the Hermes integration (gbrain as long-term memory) + a self-healing free-model fallback chain + desktop-app→VPS remote + local live voice, with all secrets externalized and all third-party boundaries documented. | EXPLICIT | SRC-002 |
| Current Alternatives | CAN-004 | (a) Bespoke manual setup per host (status quo, what this replaces); (b) cloud-hosted agent-memory SaaS (not self-hosted, not free); (c) running Hermes purely local with no shared/persistent VPS brain. | ASSUMPTION | SRC-001 |
| Key Capabilities | CAN-005 | 1) gbrain backend (MCP+PGLite) + gbrain-atlas frontend via Docker; 2) Hermes-config overlay: `config.yaml.template` + `gbrain_peek.sh`/`gbrain_promote.sh` hooks + `SOUL.md.template`; 3) model-health watcher + cron (auto-replace pulled OpenRouter models); 4) VPS systemd unit templates + `vps/bootstrap.sh` + portable privacy gate; 5) local Mac glue: SSH tunnel, Ollama-over-Tailscale, desktop remote/local launchers, atlas, LaunchAgents; 6) local voice: one-shot PTT + vault MCP server. | EXPLICIT | SRC-002 |
| Non-Goals | CAN-006 | Redistributing the Nous Hermes agent/desktop app (third-party); shipping a working calendar broker (documented only — its VPS side is an SSH forced-command wrapper); full feature parity on single-machine or non-Mac setups; managing the user's Nous account or secrets. | EXPLICIT | SRC-002 (THIRD-PARTY.md), SRC-001 |
| Constraints | CAN-007 | Hermes is third-party → user installs it with their own Nous account. Topology is two-machine (VPS + Mac + Tailscale), opinionated. The reference OpenRouter account has **zero credits** → free models only. VPS must be AMD64 (ARM images from a Mac won't run). Voice surface is macOS-only (say/sox/whisper/Hammerspoon). Secrets are never committed (`.env`, `*.token`, `connection.json`, `auth.json` gitignored). | EXPLICIT | SRC-001, SRC-005 |
| Risks | CAN-008 | RISK-001 Hermes config schema drift (`_config_version`) rots `config.yaml.template`. RISK-002 free-model 429/404 flakiness (mitigated by the watcher, but the curated PRIORITY list goes stale). RISK-003 ARM/AMD docker mismatch. RISK-004 gateway runs as **root** (security tradeoff — hermes home under `/root`). RISK-005 if an adopter builds the calendar broker, the SSH forced-command wrapper is a footgun. RISK-006 no end-to-end deploy has been verified on a clean host (only component-level live checks this session). | EXPLICIT | SRC-001, SRC-005 |
| Success Signal | CAN-009 | Same as VIS-006: a fresh adopter reaches a working stack from the docs alone — gbrain MCP up, Hermes using it, fallback chain valid + self-healing, desktop→VPS, one PTT turn. | EXPLICIT | SRC-001 |
| Evidence | CAN-010 | Verified live THIS session (component level): OpenRouter `/models` returns 337, the chosen models return HTTP 200, gbrain MCP `initialize` handshake returns serverInfo gbrain v0.42.48.0, watcher `--dry-run` reports all alive + Telegram notify confirmed, all 4 VPS services active, model-health cron installed. NOT yet verified: a clean end-to-end deploy on a fresh VPS+Mac, ARM/AMD build. | EXPLICIT | SRC-001 |
| Allowed Scope | CAN-011 | The kit files already merged to `main`: root (`install.sh`, `bootstrap.sh`, `.env.example`, `docker-compose.yml`, `Dockerfile.*`, `LICENSE`, `THIRD-PARTY.md`, `README.md`, `run_all.sh`), `vps/` (systemd units + drop-ins + `bootstrap.sh` + `scripts/{model_health_check.py,install-cron.sh,privacy-gate.sh}`), `hermes-config/` (`config.yaml.template`, `SOUL.md.template`, `.env.example`, `hooks/`), `local/` (tunnel/ollama/desktop/atlas/ptt-turn/vault-mcp/voice.config + `launchagents/` + `bootstrap.sh` + `connection.json.template` + `ssh-config-snippet`), `docker/nginx.conf`, `docs/` (topology, tailscale, hermes-install, voice, troubleshooting). Plus the cloned-not-vendored `backend/` (garrytan/gbrain) + `frontend/` (DYAI2025/gbrain-atlas). | EXPLICIT | SRC-002 |
